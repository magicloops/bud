# get.bud.dev Release Hosting

## Purpose

`https://get.bud.dev` is the public host for the Bud daemon installer and
release artifacts.

Phase 3 produces the artifact bundle and manifest in GitHub Actions. The actual
host upload is still an operator/deployment step until the production hosting
backend and credentials are wired into CI.

## Required Paths

Versioned artifacts are immutable:

```text
/releases/vX.Y.Z/bud-aarch64-apple-darwin.tar.gz
/releases/vX.Y.Z/bud-x86_64-apple-darwin.tar.gz
/releases/vX.Y.Z/bud-x86_64-unknown-linux-gnu.tar.gz
```

The stable channel manifest is mutable and points at immutable artifacts:

```text
/releases/stable/manifest.json
```

The installer path is owned by Phase 4:

```text
/install.sh
```

## Manifest Contract

The manifest is generated by `scripts/bud-release.mjs` from actual artifact
metadata and has this shape:

```json
{
  "version": "vX.Y.Z",
  "channel": "stable",
  "published_at": "2026-05-30T00:00:00Z",
  "artifacts": [
    {
      "target": "x86_64-unknown-linux-gnu",
      "url": "https://get.bud.dev/releases/vX.Y.Z/bud-x86_64-unknown-linux-gnu.tar.gz",
      "sha256": "...",
      "min_os": "glibc 2.35",
      "size": 12345678
    }
  ]
}
```

Installers must verify the downloaded archive SHA-256 before extracting it.

## V1 Integrity Policy

Minimum v1 artifact integrity:

- HTTPS-only artifact and manifest delivery
- immutable versioned artifact URLs
- SHA-256 checksums generated from CI outputs
- stable manifest generated from the same CI output metadata

Deferred before broader public launch:

- macOS code signing
- macOS notarization
- provenance/attestation publishing
- automated vulnerability audit gate

## Upload Handoff

Until CI has production hosting credentials, a release operator should:

1. Run the `Bud Release Artifacts` workflow from a tag or explicit manual run.
2. Download the uploaded target archives and `bud-release-manifest` artifact.
3. Upload archives to `/releases/<version>/`.
4. Upload the manifest to `/releases/stable/manifest.json`.
5. Confirm every manifest URL is reachable and the SHA-256 matches the hosted
   bytes before exposing the corresponding installer path.
