# v1

Version 1 Bud daemon-service protobuf schema.

## Files

### `bud.proto`

Defines `BudControl.Connect`, `BudData.Attach`, `BudEnvelope`, traffic/transport enums, typed errors, operation and stream lifecycle enums, reconnect/reconciliation payloads, current terminal/control payloads, data-plane attach/stream payloads, Phase 4.2 proxy-open payloads, and a `LegacyJsonPayload` compatibility payload retained for downgrade/fixture support.

Known current payload messages include a transitional `frame_json` field so WebSocket-capable peers can dispatch by typed oneof payload tag before the service and daemon switch fully to generated field-level protobuf structs.

`BudData.Attach` is the Phase 3 HTTP/2 data fallback service. The initial implementation uses it for daemon-to-service `terminal_output` while keeping lifecycle, auth, heartbeat, terminal requests, and request-scoped terminal results on `BudControl.Connect`. Phase 4 adds runtime handling for generic `stream_data`, `stream_credit`, `stream_reset`, and `stream_close` frames so proxy/file work can build on bounded data-only streams. Phase 4.2 adds `proxy_open` and `proxy_open_result` typed payload tags for daemon-backed localhost HTTP proxy streams. Phase 4.4 adds `file_open` and `file_open_result` typed payload tags for read-only workspace file stat/read/range streams.
