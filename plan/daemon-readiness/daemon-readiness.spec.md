# daemon-readiness

Implementation planning documents for shipping the Rust Bud daemon as a production binary with one-command install, host dependency preflight, service-generated claim ownership, release artifacts, user-service management, and clean-machine validation.

## Purpose

This folder turns the production-readiness review in:

- [../../review/bud-daemon-production-binary-readiness.md](../../review/bud-daemon-production-binary-readiness.md)

into an actionable phased implementation plan.

The plan assumes:

- `https://api.bud.dev` is the production API origin
- `wss://api.bud.dev/ws` is the expected daemon WebSocket URL unless the route changes
- `https://get.bud.dev` hosts the installer and release artifacts
- v1 production installs are user-scoped machine installs
- launchd and systemd user services are the managed background targets
- foreground mode remains available for development and unsupported systems
- tmux is a required host dependency for terminal support, not bundled and not silently installed
- authenticated install commands are generated by the service and carry a 10 minute single-use claim/install identifier

Service and daemon claim phase covering:

- 10 minute single-use install claims

- QR/link fallback preservation
- schema/migration requirements
- ownership validation

Release phase covering:

- platform artifact matrix
- CI release builds
- hosted manifest
- SHA-256 verification

- signing/provenance policy
- source-package/protobuf build concerns

Detailed GitHub Release publication, Cloudflare Worker, installer script,
promotion, and fallback work is scoped under

Installer phase covering:

- `install.sh`

- `~/.bud/bin/bud` install path
- missing tmux handling
- launchd user agent setup
- systemd user service setup
- foreground fallback
- upgrade/uninstall behavior

### `phase-5-web-install-surface-and-docs.md`

Web/docs phase covering:

- public tokenless command
- authenticated generated command UI
- claim expiration/redeemed states
- QR/link fallback
- capability disclosure
- install/update/uninstall/status docs

### `phase-6-validation-rollout-and-follow-ups.md`
