# migrations

SQL migration files generated by Drizzle Kit for schema evolution.

## Purpose

Contains sequential SQL migrations that transform the database schema from one version to the next. Managed by `drizzle-kit` and applied via `pnpm db:migrate`.

## Files

### `.gitkeep`

Placeholder to ensure the directory exists in git.

### `0000_opposite_morbius.sql`

**Initial schema** - Creates base tables:
- `bud` - Device registry
- `enrollment_token` - Registration tokens
- `thread` - Conversations
- `message` - Chat messages
- `run` - Command execution records
- `run_step` - Tool call steps
- `run_log` - Stdout/stderr storage
- `run_summary` - Denormalized summaries

### `0001_stiff_magdalene.sql`

**Legacy sessions** - Adds PTY session support:
- `session` - Session records with backend type
- `session_log` - Session output storage
- Updates to `thread` for session linking

### `0002_glorious_james_howlett.sql`

**Terminal sessions** - Initial terminal infrastructure:
- `terminal_session` - Thread-scoped session records
- `terminal_session_output` - Output chunk storage
- Indexes for efficient querying

### `0003_thread_current_session.sql`

**Thread-session linking**:
- Adds `current_session_id` column to `thread` table
- Index for session lookup

### `0004_smart_joystick.sql`

**Terminal session enhancements**:
- Additional columns for `terminal_session`:
  - `cols`, `rows` - Terminal dimensions
  - `total_output_bytes` - Cumulative output size
  - `idle_timeout_minutes`, `max_lifetime_hours`
- `terminal_session_input_log` - Input audit trail

### `0005_terminal_output_pk_byte_offset.sql`

**Output storage optimization**:
- Changes primary key of `terminal_session_output`
- Adds `byte_offset` for efficient seeking
- Optimizes indexes for tail queries

### `0006_terminal_sessions.sql`

**Thread-scoped sessions completion**:
- Adds `thread_id` to `terminal_session`
- Foreign key to `thread` table
- State machine column (`pending`, `creating`, `ready`, etc.)
- Soft delete support via `closed_at`

## Migration Naming

Drizzle Kit generates names in format: `{sequence}_{adjective}_{noun}.sql`

The adjective-noun pairs are randomly generated and have no semantic meaning.

## Subfolder

### `meta/`

Drizzle Kit metadata tracking migration state. Contains:
- `_journal.json` - Migration history
- Snapshot files for each migration

## Usage

```bash
# Generate migration from schema changes
pnpm db:generate

# Apply pending migrations
pnpm db:migrate

# Push schema directly (dev only)
pnpm db:push
```

## Schema Evolution

```
v0: bud, thread, message, run (base)
 │
 ▼
v1: + session, session_log (legacy PTY)
 │
 ▼
v2: + terminal_session, terminal_session_output (tmux)
 │
 ▼
v3-5: terminal session refinements
 │
 ▼
v6: thread-scoped sessions (current)
```
